## 你需要做的事（共 5 步）

### 第 1 步：拿三个 API Key（约 20 分钟）

#### 1a. Claude API Key（Anthropic）

1. 访问：https://console.anthropic.com/
2. Sign Up → 用 Google 登录最快
3. Billing → Add Credits → 充 **$10**
4. API Keys → Create Key → 名字填 `news-map`
5. **复制 key**（以 `sk-ant-` 开头）

#### 1b. 高德地图 Key

1. 访问：https://lbs.amap.com/（国内可访问）
2. 用手机号注册 + 实名认证
3. Console → 应用管理 → 创建新应用
4. 应用名 `中亚新闻地图`
5. 添加 Key → 服务平台选 **Web 端 (JS API)**
6. **复制 key**（32 位字符）

#### 1c. GitHub 账号

1. 访问：https://github.com/
2. Sign Up（如果没有）

把这三样东西先存到桌面 `keys.txt` 里：
```
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
AMAP_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
GitHub 用户名: xxx
```

---

### 第 2 步：把代码推到 GitHub（约 5 分钟）

#### 2a. 在 GitHub 网页创建仓库

1. 登录 GitHub
2. 右上角 + → New repository
3. Repository name：`central-asia-news-map`
4. **Public**（用免费版必须 Public）
5. **不要勾任何初始化选项**（README、.gitignore、License 都不勾）
6. Create repository

完成后 GitHub 会显示一个页面，里面有几行命令。**先不要按那个执行**，照下面来。

#### 2b. 在你电脑终端推送代码

打开 iTerm2（如果没装就用系统自带的"终端"），逐条复制粘贴执行（**把 `你的GitHub用户名` 换成实际用户名**）：

```bash
cd ~/Desktop/中亚新闻地图/code
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的GitHub用户名/central-asia-news-map.git
git push -u origin main
```

第一次 push 时它会要登录：
- 用户名：你的 GitHub 用户名
- 密码：**不是你的 GitHub 密码**！是 Personal Access Token

如果它提示要密码，按这里生成 token：
1. https://github.com/settings/tokens
2. Generate new token (classic)
3. 名字填 `news-map`，过期时间选 90 days
4. 勾上 `repo` 全部
5. 生成，复制（以 `ghp_` 开头）
6. 在终端密码处粘贴这个 token

完成后访问 `https://github.com/你的用户名/central-asia-news-map`，能看到 `backend`/`frontend` 等文件夹就成功了。

---

### 第 3 步：在 Render 部署（约 5 分钟）

#### 3a. 注册 Render

1. 访问：https://render.com/
2. Sign Up → **选 "GitHub"**（重要！这样它能直接读你的仓库）
3. 授权 Render 访问你的 GitHub

#### 3b. 创建 Blueprint（一次性创建后端 + 前端）

1. 登录 Render
2. 左侧 + → "Blueprint"
3. Connect a repository → 选 `central-asia-news-map`
4. Render 会读取仓库里的 `render.yaml` 自动识别两个服务
5. Blueprint Name：`central-asia-news-map`
6. 点 "Apply"

Render 开始构建。**会失败**——因为还没填环境变量，正常。

---

### 第 4 步：填环境变量（约 5 分钟）

#### 4a. 后端环境变量

1. Render 主页 → 左侧 Services → 点 `news-map-api`
2. 左侧 "Environment"
3. 找到 `ANTHROPIC_API_KEY`，点旁边的 Edit
4. Value 粘贴你的 `sk-ant-xxx` key
5. Save

保存后 Render 会自动重新部署。

#### 4b. 拿到后端的 URL

回到 `news-map-api` 服务页面顶部，能看到一个 URL 类似：
```
https://news-map-api-xxxx.onrender.com
```

**复制这个 URL**。

#### 4c. 前端环境变量

1. 回到主页 → Services → 点 `news-map-frontend`
2. 左侧 "Environment"
3. 找到 `AMAP_KEY`，Edit → 粘贴你的高德 key
4. 找到 `API_BASE`，Edit → 粘贴上面后端的 URL（**完整带 https://，结尾不要加 /**）
5. Save

保存后前端也会重新部署。

---

### 第 5 步：访问网站（约 1 分钟）

1. Render 主页 → Services → 点 `news-map-frontend`
2. 顶部能看到前端 URL，类似 `https://news-map-frontend-xxxx.onrender.com`
3. **打开 VPN**
4. 浏览器访问这个 URL

应该能看到：
- 深色背景的地图
- 中亚区域有 5-10 个彩色点位
- 点击点位弹出新闻

⚠️ **第一次访问慢是正常的**：
- 后端首次启动要 30-60 秒
- 之后就正常了

## 如果遇到问题

### 后端构建失败

Render 主页 → news-map-api → Logs，把红色错误信息发给我。

### 前端构建失败

通常是没填环境变量。检查 news-map-frontend 的 Environment，三个值都要填。

### 网站打开但地图空白

按 F12 打开浏览器控制台，看 Console 标签的红字。常见原因：
- 高德 key 没启用 "JS API" → 去高德控制台检查
- API_BASE 填错（多了 `/` 或少了 `https://`）

### 地图正常但没有新闻点

后端还在抓数据。等几分钟刷新。如果 30 分钟后还没有：
- `news-map-api` → Logs 看抓取日志
- 把日志发我

### 把任何错误发我

**完整复制错误信息**（不要只描述），我能 5 分钟解决。
