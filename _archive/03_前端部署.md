
**修改 2**：把模拟数据改成调真实后端

找到这一段（在 `<script>` 标签里靠下的位置）：

```javascript
function init() {
  if (typeof AMap === 'undefined') {
    ...
  }
  initMap();
  setTimeout(() => {
    applyFilter();
    document.getElementById('loadingOverlay').classList.add('hidden');
  }, 600);
}
```

替换成：

```javascript
const API_BASE = 'http://你的服务器IP:8000';

let realNews = [];

async function loadNews() {
  try {
    const resp = await fetch(`${API_BASE}/api/news?days=7&limit=300`);
    if (!resp.ok) throw new Error('HTTP ' + resp.status);
    const data = await resp.json();
    realNews = data.items.map(n => ({
      id: n.id,
      country: n.country,
      city: n.city,
      lat: n.lat,
      lng: n.lng,
      title_zh: n.title_zh,
      summary_zh: n.summary_zh,
      source: n.source_name,
      source_url: n.source_url,
      published_at: n.published_at
    }));
    document.getElementById('updateInfo').textContent = '🔄 更新于刚刚';
  } catch (e) {
    console.error('加载新闻失败:', e);
    document.getElementById('updateInfo').textContent = '⚠ 数据加载失败';
    realNews = [];
  }
}

function init() {
  if (typeof AMap === 'undefined') {
    document.getElementById('loadingOverlay').innerHTML = `
      <div style="color: var(--text-secondary); text-align: center;">
        <div style="font-size: 32px; margin-bottom: 12px;">🗺</div>
        <div>地图加载失败</div>
        <div style="font-size: 12px; margin-top: 8px;">请检查高德 API Key 配置</div>
      </div>`;
    return;
  }
  initMap();
  loadNews().then(() => {
    applyFilter();
    document.getElementById('loadingOverlay').classList.add('hidden');
  });
}
```

⚠️ 把 `你的服务器IP` 替换为你 Vultr 服务器的真实 IP。

**修改 3**：原来 `applyFilter` 用的是 `MOCK_NEWS`，改成 `realNews`

找到：
```javascript
function applyFilter() {
  const items = currentFilter === 'ALL'
    ? MOCK_NEWS
    : MOCK_NEWS.filter(n => n.country === currentFilter);

}
```

把两处 `MOCK_NEWS` 都改成 `realNews`。

同样找到 `focusNews` 函数里的 `MOCK_NEWS.find(...)`，也改成 `realNews.find(...)`。

## 步骤 2：部署到 Vercel

### 2.1 装 Vercel CLI

iTerm2 执行：

```bash
brew install node
```

如果没装 brew，先装 brew（按提示走）：
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

装好 node 后：
```bash
npm install -g vercel
```

### 2.2 部署

```bash
cd ~/Desktop/news-map-frontend
vercel
```

第一次会问几个问题：
- Set up and deploy? → `Y`
- Which scope? → 选你的账号
- Link to existing project? → `N`
- What's your project's name? → `central-asia-news`（或随便起）
- In which directory is your code located? → `./`（直接回车）
- Want to modify these settings? → `N`

部署 30 秒后会输出一个 URL，类似：
```
https://central-asia-news-xxxx.vercel.app
```

🎯 **完成标志**：浏览器打开这个 URL，能看到你的网站。

## 步骤 3：解决 HTTPS / HTTP 混合内容问题

⚠️ 你可能在浏览器控制台（F12 打开）看到错误：
```
Mixed Content: The page at 'https://...' was loaded over HTTPS,
but requested an insecure resource 'http://你的IP:8000/api/news'.
```

**原因**：Vercel 默认是 HTTPS，但你的后端是 HTTP，浏览器不允许混用。

### 解决方案：给后端加 HTTPS

我们用 Caddy（最简单的反向代理），自动配 HTTPS。

但 Caddy 需要域名才能配 HTTPS。所以你有两个选择：

#### 方案 A（最简单）：买个域名

1. 在 Namecheap、Cloudflare 或者 Porkbun 买个便宜域名（约 $10/年）
2. 域名 DNS 设 A 记录指向你的服务器 IP
3. SSH 到服务器，装 Caddy：

```bash
apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | tee /etc/apt/trusted.gpg.d/caddy-stable.asc
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | tee /etc/apt/sources.list.d/caddy-stable.list
apt update
apt install caddy
```

4. 配置 Caddy：
```bash
nano /etc/caddy/Caddyfile
```

把内容改为（替换为你的真实域名）：
```
api.你的域名.com {
    reverse_proxy localhost:8000
}
```

5. 重启 Caddy：
```bash
systemctl restart caddy
```

Caddy 自动申请 Let's Encrypt 证书，几秒钟就好。

6. 修改前端 `index.html`，把 API_BASE 改成：
```javascript
const API_BASE = 'https://api.你的域名.com';
```

7. 重新部署前端：
```bash
cd ~/Desktop/news-map-frontend
vercel --prod
```

#### 方案 B（最快但不太正规）：把前端也改成 HTTP

不推荐，但 MVP 阶段可以：

把前端也部署到 HTTP 而非 Vercel。最简单的是：
- 在服务器上用 Caddy 直接把 HTML 也托管了
- 然后访问 `http://你的IP/` 而不是 Vercel 域名

```bash
mkdir -p /var/www/html
nano /etc/caddy/Caddyfile
```

内容：
```
:80 {
    handle /api/* {
        reverse_proxy localhost:8000
    }
    handle {
        root * /var/www/html
        file_server
    }
}
```

然后把 `index.html` 上传到 `/var/www/html/`：
```bash
# 在你本地电脑上
scp ~/Desktop/news-map-frontend/index.html root@你的IP:/var/www/html/
```

把 `index.html` 里的 `API_BASE` 改成相对路径：
```javascript
const API_BASE = '';
```

然后重启 Caddy：
```bash
systemctl restart caddy
```

访问 `http://你的IP/` 就是网站。

**优先方案 A，花 $10 买个域名值得**。MVP 阶段也能让你练习一遍正经部署流程。
