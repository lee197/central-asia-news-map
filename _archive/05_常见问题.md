# 05 · 常见问题排错手册

遇到问题先在这里找答案，找不到就把错误信息发给我。

---

## 通用排错原则

### 三步走

1. **看错误信息**：哪一步报的，红字写的什么
2. **查这份文档**：搜关键词
3. **发给我**：把完整错误粘贴出来

### 遇到错误不要慌

每个工程师都会遇到一万次错误。错误是反馈，不是失败。

把错误原样复制给我，**不要自己描述**。比如：
- ❌ "它好像连不上"
- ✅ "执行 `ssh root@1.2.3.4` 后报错：`ssh: connect to host 1.2.3.4 port 22: Operation timed out`"

## SSH 相关

### 问题 1：`ssh: connect to host xxx port 22: Operation timed out`

**原因**：网络不通

**排查**：
1. VPN 是否开着？
2. 服务器 IP 是不是抄错了？登录 Vultr 确认
3. 服务器是不是关机了？登录 Vultr 看状态

### 问题 2：`Permission denied (publickey,password)`

**原因**：密码不对

**排查**：
1. 是不是粘贴密码时多了空格？
2. 在 Vultr 控制台重新点眼睛图标看密码，重新复制

### 问题 3：每次 SSH 都要输密码很烦

设置免密登录：

本地终端：
```bash
ssh-keygen -t ed25519
# 一路回车
ssh-copy-id root@你的IP
# 输入密码
```

之后 `ssh root@你的IP` 就不再要密码。

## 后端相关

### 问题 4：`systemctl status news-map` 显示 failed

看日志：
```bash
tail -100 /var/log/news-map.log
```

常见原因：

#### 4.1 `RuntimeError: ANTHROPIC_API_KEY 未设置`

`.env` 文件没配好：
```bash
cat /opt/news-map/.env
```

应该看到 `ANTHROPIC_API_KEY=sk-ant-xxxx`。

如果没有或不对，重新编辑：
```bash
nano /opt/news-map/.env
```

### 问题 5：API 返回 `{"count":0,"items":[]}` 一直没数据

**原因 A**：抓取还没跑完

第一次抓取需要 5-15 分钟。看日志：
```bash
tail -f /var/log/news-map.log
```
等到 `=== 抓取完成 ===`。

**原因 B**：所有 LLM 处理都被过滤掉了

看日志里有没有：
```
LLM 处理失败: ...
```
或者大量条目被丢弃。

把日志最后 100 行发我看。

**原因 C**：Claude API key 余额用完

登录 https://console.anthropic.com/ 看 Billing 余额。

### 问题 6：Claude API 报错 `401 unauthorized`

API key 失效或写错。重新复制粘贴 key 到 `.env`：
```bash
nano /opt/news-map/.env
systemctl restart news-map
```

### 问题 7：浏览器访问 `http://IP:8000/api/news` 打不开

**排查清单**：
```bash
# 1. 服务在跑吗？
systemctl status news-map

# 2. 端口在监听吗？
ss -tlnp | grep 8000

# 3. 防火墙开了吗？
ufw status
ufw allow 8000

# 4. VPN 开了吗？（本地）
```

### 问题 8：日志看到 `RSS XXX 失败: timeout`

某个 RSS 源访问慢或被墙了。这是**正常现象**，代码已经处理了，会跳过失败的源继续抓其他的。

如果某个源持续失败超过一周，可以编辑 `app.py`，把那条 RSS 注释掉：
```python
RSS_FEEDS = [
    {"url": "...", ...},
    # {"url": "无效源", ...},  # 注释掉
]
```

## 前端相关

### 问题 9：浏览器打开网站，地图区域是空白

按 F12 打开开发者工具，看 Console（控制台）有没有红色错误。

#### 9.1 `Failed to load resource: AMap`

高德 API key 问题：
- 检查 `index.html` 里 `key=` 后面是不是真的 key
- 高德控制台看 key 是否启用了 "JS API" 服务

#### 9.2 `INVALID_USER_KEY`

key 错了或没启用 JS API。去高德控制台检查。

#### 9.3 `USERKEY_PLAT_NOMATCH`

key 平台不匹配。创建 key 时必须选 "Web 端 (JS API)"。

### 问题 10：地图能显示但没有点位

**排查**：F12 → Network 标签 → 刷新页面 → 找 `/api/news` 请求

#### 10.1 请求显示红色 / failed

后端连不上：
- VPN 开了吗？
- `index.html` 里 `API_BASE` 是不是写对了？
- `curl http://你的IP:8000/api/news` 在终端能不能访问？

#### 10.2 请求 200 但 items 是空数组

后端没数据。回去看问题 5。

#### 10.3 `Mixed Content` 红字

HTTPS / HTTP 混合内容问题。看 [03_前端部署.md 步骤 3](./03_前端部署.md)。

### 问题 11：CORS 错误：`No 'Access-Control-Allow-Origin' header`

后端代码里已经配了 `allow_origins=["*"]`，理论上不会出现。

如果出现：
```bash
ssh root@你的IP
grep "allow_origins" /opt/news-map/app.py
```
应该看到 `allow_origins=["*"]`。

如果没有，重新粘贴 02 里的代码。

## Vercel 相关

### 问题 12：`vercel` 命令报错 `command not found`

```bash
npm install -g vercel
```

如果还不行：
```bash
which node
node --version
```
检查 node 是否安装。如果没装：
```bash
brew install node
```

### 问题 13：Vercel 部署后访问 404

`index.html` 文件名是不是大写了？必须全小写：`index.html`。

```bash
cd ~/Desktop/news-map-frontend
ls -la
```

## 费用相关

### 问题 14：Anthropic 账单突然飙升

立刻停服务：
```bash
ssh root@你的IP
systemctl stop news-map
```

然后查日志找异常调用：
```bash
grep "LLM" /var/log/news-map.log | tail -100
```

通常是：
- 代码 bug 导致死循环
- RSS 源返回了一万条新条目（超大）

修复后重启：
```bash
systemctl start news-map
```

### 问题 15：Vultr 账户余额不足

登录 Vultr → Billing → Add Funds，再充 $10。

⚠️ 余额不足 Vultr 会**直接删掉服务器**！设置自动扣款或者每月看一次余额。

## 数据相关

### 问题 16：想手动添加一条新闻测试

SSH 到服务器：
```bash
sqlite3 /opt/news-map/news.db
```
进入 SQLite 命令行：
```sql
INSERT INTO news (id, source_url, source_name, country, city, lat, lng,
                  title_orig, title_zh, summary_orig, summary_zh,
                  published_at, confidence, lang_orig)
VALUES ('test001', 'https://example.com', 'Test',
        'UZ', '塔什干', 41.2995, 69.2401,
        'Test', '测试新闻',
        'Test summary', '这是一条测试新闻',
        '2026-04-30T00:00:00Z', 0.9, 'en');
.quit
```

刷新网站应该能看到。

### 问题 17：想清空数据库重新抓

```bash
ssh root@你的IP
systemctl stop news-map
rm /opt/news-map/news.db
systemctl start news-map
```

下一次定时抓取会重建。

### 问题 18：想看数据库里有什么

```bash
sqlite3 /opt/news-map/news.db
.headers on
.mode column
SELECT country, city, title_zh FROM news ORDER BY published_at DESC LIMIT 10;
.quit
```

## 想改东西

### 问题 19：想改网站标题、配色

直接编辑本地的 `index.html`，然后：
```bash
cd ~/Desktop/news-map-frontend
vercel --prod
```

### 问题 20：想加新的 RSS 源

编辑服务器上的 `app.py`：
```bash
nano /opt/news-map/app.py
```

找到 `RSS_FEEDS` 列表，添加新的源：
```python
RSS_FEEDS = [
    ...
    {"url": "新源URL", "country": "KZ", "source": "新源名"},
]
```

保存后重启：
```bash
systemctl restart news-map
```

### 问题 21：想改抓取频率

编辑 `app.py`，找到：
```python
scheduler.add_job(run_ingest, "interval", minutes=30, ...)
```

改 `minutes=30` 为你想要的值（比如 60）。重启服务。

1. **你想做什么**（一句话）
2. **完整的错误信息**（截图或文字）
3. **你执行的具体命令**
4. **你已经试过什么**

> 报错：`curl: (7) Failed to connect to 1.2.3.4 port 8000: Connection refused`
> 我已经检查过 `systemctl status news-map` 显示 active running。

这样的描述我能 5 分钟给你解决方案。
