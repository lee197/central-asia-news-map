## 步骤 1：验证定时抓取

后端代码里已经设置了：
- 每 30 分钟自动抓一次新闻
- 每天凌晨 3 点清理 30 天前数据

### 1.1 看日志确认调度器在跑

SSH 到服务器：
```bash
tail -f /var/log/news-map.log
```

应该能看到日志，每 30 分钟出现一次：
```
=== 开始抓取 ===
RSS Tengrinews: 拿到 30 条
...
=== 抓取完成，新增 X 条 ===
```

按 `Ctrl + C` 退出查看。

### 1.2 手动触发一次抓取（可选）

如果你不想等 30 分钟：

```bash
cd /opt/news-map
source venv/bin/activate
export $(cat .env | xargs)
python3 -c "
import asyncio
from app import run_ingest, init_db
init_db()
asyncio.run(run_ingest())
"
```

跑 5-10 分钟，结束后再查 API：
```bash
curl http://localhost:8000/api/health
```

应该看到 `total_news` 增加了。

🎯 **完成标志**：日志显示定期抓取在跑，total_news 数字在增加。

## 步骤 2：测试服务器重启自动恢复

模拟服务器突然重启的情况。

### 2.1 重启服务器

在服务器 SSH 终端：
```bash
reboot
```

SSH 会断开。等 1 分钟。

### 2.2 重新连接

```bash
ssh root@你的IP
```

### 2.3 检查服务是否自动起来了

```bash
systemctl status news-map
```

应该看到 `Active: active (running)`。

浏览器访问 `http://你的IP:8000/api/health`，应该正常返回。

🎯 **完成标志**：重启服务器后，5 分钟内服务自动恢复，网站能正常访问。

## 步骤 3：简单的健康监控

最简单的监控：你每天上网时手动看一眼 `/api/health` 接口。

更好的做法：用 UptimeRobot（免费）做监控。

### 3.1 注册 UptimeRobot

1. 访问：https://uptimerobot.com/
2. 注册（免费版够用）
3. 登录后点 "+ Add New Monitor"
4. 设置：
   - Monitor Type：HTTP(s)
   - Friendly Name：中亚新闻地图
   - URL：`http://你的IP:8000/api/health`（或域名）
   - Monitoring Interval：5 分钟
5. 通知方式：邮箱（默认）
6. 保存

### 3.2 测试

UptimeRobot 会立刻测一次，正常的话 1 分钟内显示绿色 "Up"。

如果你的服务器挂了，5 分钟内你会收到邮件通知。

🎯 **完成标志**：UptimeRobot 显示 Up，并且收到了"网站正常上线"的测试邮件。

## 步骤 4：备份重要文件

万一服务器被删了，你能快速恢复。

### 4.1 备份代码和密钥

在你**本地电脑**：
```bash
mkdir -p ~/Desktop/news-map-backup
cd ~/Desktop/news-map-backup
scp root@你的IP:/opt/news-map/app.py .
scp root@你的IP:/opt/news-map/.env .
scp root@你的IP:/opt/news-map/start.sh .
scp root@你的IP:/etc/systemd/system/news-map.service .
```

### 4.2 备份数据库（每周做一次）

```bash
scp root@你的IP:/opt/news-map/news.db ~/Desktop/news-map-backup/news-$(date +%Y%m%d).db
```

🎯 **完成标志**：桌面有 `news-map-backup` 文件夹，包含上述文件。

## 步骤 5：成本监控

每周看一次账单，避免意外支出。

### 5.1 Vultr 账单

登录 Vultr → Billing。每月固定 $6 左右。

### 5.2 Anthropic 账单

登录 https://console.anthropic.com/ → Billing。

按你的用量，每天约 $0.2-0.5，每月约 $5-15。

⚠️ 如果某天突然飙升（比如 $5），说明代码有 bug 在循环调用。立刻 SSH 到服务器执行：
```bash
systemctl stop news-map
```
然后查日志找问题。

## 上线后日常使用

### 看数据
- 浏览器开 VPN 访问网站
- 或者 `curl http://你的IP:8000/api/news | head` 看接口

### 看日志
SSH 到服务器：
```bash
tail -f /var/log/news-map.log
```

### 重启服务
如果觉得服务有问题：
```bash
systemctl restart news-map
```

### 改代码
1. 在本地 VS Code 改 `app.py` 备份
2. 上传到服务器：
   ```bash
   scp app.py root@你的IP:/opt/news-map/app.py
   ```
3. 重启：
   ```bash
   ssh root@你的IP "systemctl restart news-map"
   ```
